# ADR-0035 — Outcome-capture endpoint: public Lambda Function URL + HMAC token

**Status:** Accepted · **CRUCIAL** · merged PR #34 → **deployed + live-validated 2026-07-20**. Squad-built from [INV-001](../investigations/INV-001-dark-feedback-loop/README.md) (Investigator → Surgeon → Examiner **CLEAN PASS**).

## Context
[INV-001](../investigations/INV-001-dark-feedback-loop/README.md) verified the **dark feedback loop** live: `application_event` had **0 rows** and `score_override` was set on **1/286** — the tool scores jobs, emails them, and never learns whether a "90" led to an application, so scoring can't be measured and **M7 calibration has nothing to calibrate toward.** The capability to record outcomes existed (`scripts/track.py`, the v0.12.0 panel Curate tab) but was **CLI-/app-only** — the user applies *from the email in their browser* and won't context-switch to a terminal. Root cause: **no capture affordance where the user already is.** Rung 2 (a capture endpoint the email/report links hit) was approved at Checkpoint A.

## Decision
A **public AWS Lambda Function URL** (`authorization_type = "NONE"`) fronting **one new Lambda** (the same deployment zip, handler `jobfetcher.handlers.capture.handler`), where **the auth is a token, not the network**: every request must carry a short-lived **HMAC-signed token** ([`core/capture_token.py`](../../src/jobfetcher/core/capture_token.py)) scoped to exactly `{posting_id, status}` with a **30-day TTL**. The signing key is **generated and owned by Terraform** in Secrets Manager (a self-owned secret, not a third-party credential). The digest + full-list report inject per-job "✓ Mark applied" links via an **injected `capture_link` callable** (the renderers stay pure; a missing base-URL/key degrades to **no link**). `verify` runs in constant time **before any DB touch**, so a forged/expired/tampered token returns 400 with **zero rows written**; a valid token drives exactly ONE write through the existing **`Repository.track_application_event`** (the append-only `application_event`, [ADR-0026](0026-outcome-tracking-override-lineage.md)). Least-privilege IAM (Data-API + its two secrets + Logs only), the key never in outputs/logs. **No DB migration, no new runtime dependency** (the `random` provider is Terraform-only). _(A `reserved_concurrent_executions=5` abuse cap was intended but **removed at deploy** — the account's total concurrency limit is 10 and AWS requires ≥10 unreserved, so any reservation is rejected; the account ceiling is the de-facto cap.)_

## Alternatives Considered
- **API Gateway + a Lambda authorizer / Cognito.** Rejected as heavier — an extra service + an authorizer + more IAM — for a single-user capture link whose auth *is* the signed token. A Function URL + token is the minimal shape. API Gateway is the **documented scale-up** if routing / throttling / WAF is ever needed.
- **An open (no-auth) write endpoint.** Rejected: anyone could forge outcome rows and corrupt the ground truth this feature exists to create.
- **Reuse the v0.10.0 presigned-S3 link.** Presigning authorizes an S3 *object read*, not a *write* endpoint; the HMAC token borrows that pattern's short-lived-signed-link ergonomics without needing an S3 object.
- **A confirmation interstitial / POST-only flow now** (defeats email-scanner GET-prefetch). **Deferred** — Checkpoint B decided to ship the true one-click GET; the interstitial is the **documented fast-follow** if prefetch bots ever record spurious rows (append-only + override makes a stray row correctable).
- **A hosted dashboard with inline outcome actions** (B-1 rung 3). The larger end-state; this is the minimal one-click surface that closes the loop now.

## Consequences
- **Closes the loop:** one click from the inbox lands an `application_event` row → scoring finally gets ground truth → **unblocks M7 calibration** (supplies the labels; calibration is its own later unit).
- **A new public surface** — the project's first internet-facing *write* endpoint — with a **bounded blast radius**: the token is single-status, posting-scoped, and expiring; `verify` precedes any DB touch (so spam never reaches Aurora); IAM is least-privilege; the key is never exposed; the account's 10-execution concurrency ceiling caps abuse (a per-function reservation is blocked by that same low limit — revisit after a limit increase).
- **Accepted tradeoff:** a one-click **GET** can be pre-fetched by an email security scanner / preview bot → a spurious `applied`. Correctable via the append-only log + override; the interstitial fast-follow is documented. (Examiner + CodeRabbit both flagged it; human-adjudicated at Checkpoint B.)
- **Feeds the dossier system:** the as-built is recorded in the [INV-001 Resolution](../investigations/INV-001-dark-feedback-loop/README.md#resolution--as-built-filled-at-close-when-the-fix-ships).
