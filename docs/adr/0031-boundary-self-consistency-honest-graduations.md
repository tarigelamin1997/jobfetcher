# ADR-0031 — Boundary self-consistency + honest graduations

**Status:** Accepted · **merged (PR #31, 2026-07-11)**, pending live-validation + release tag · the P2-scan winner after v0.10.0, built by the agentic squad (Investigator → Surgeon → Examiner **CLEAN PASS**) · **CRUCIAL tier** (scoring semantics — both human checkpoints honored: the [§48 scan brief](../../CLAUDE.md) approved before code, the PR approved before merge)

## Context
The scorer's holistic number is produced by a **non-deterministic LLM at temp 0** (DeepSeek `v4-pro`). A fresh P2 data-quality scan (2026-07-11, read-only on the live stack, profile held static ⇒ pure noise) measured the drift against the shortlist cutoff: **avg same-input spread 15.95 pts, max 60**, and **62 of 286 scores sat within ±16 of the threshold (60)** — i.e. roughly the *entire* ~61-job shortlist boundary was a coin-flip on membership from one run to the next. Worse, the flagship reassess **"graduation" badge was firing on that noise**: 15 measured "graduations" (jumping 15–53 pts) with an **unchanged `profile_hash`** — 100% false positives, since `_is_graduation` / the reassess counter only checked `previous_score < threshold <= score`, never whether the profile actually changed.

The scan **overturned** the roadmap agent's initial pick (the M7 shadow-`code_total` cut-over): the shadow total's own same-input spread is avg 8.11 but **max 71** — a *worse* tail than the LLM holistic — because the 7 subscores it sums come from the *same* non-deterministic LLM. The noise is in the subscores, not the weights; a deterministic re-weighting cannot remove it. M7 stays parked ([ADR-0028](0028-scorer-subscores-shadow.md) shadow instrument unchanged).

## Decision
Two changes, no migration / no infra / no new dependency.

**1 — Boundary resample (root fix; `core/ingest.py::_score_with_resample`).** Score once; **only** when the first score lands within `RESAMPLE_TRIGGER_MARGIN` of the runtime threshold (the ambiguous zone, checked on *both* sides of the cut) re-score to `RESAMPLE_N` **total** samples and keep the **median sample** — a coherent `ScoreResult` (its strengths/gaps/assessment/subscores are the ones that produced its number), never an averaged frankenscore (`_median_sample` returns the lower-median index, so an even/short count still yields a real sample). A clearly-in / clearly-out first score, or `RESAMPLE_N <= 1`, returns after **exactly one call** — the cost guard (resampling pays N× only on the ~1/5 boundary jobs, ~pennies). Wired into **both** `score_gold` and `reassess`. `RESAMPLE_N` (default **3**, odd → exact median) and `RESAMPLE_TRIGGER_MARGIN` (default **16** = the measured avg spread, deliberately wider than `near_miss_band` and an *independent* technical knob) are **module constants exposed as defaulted orchestrator params — never a required `SearchSpec` field**, so there is no config-first breaking change and `N=1` is byte-identical to the pre-resample behavior. The `Scorer` class stays pure. **Deadline-aware (H-2 "never times out"):** the first sample always runs, but each *extra* sample is gated on `deadline.expired` — past the wall we stop and return the median-so-far, never starting a new score past the Lambda timeout, so a full-set reassess can never overrun.

**2 — Honest graduations (rider; `core/notifier.py` + `core/ingest.py` + `core/ports.py` + repository).** A threshold crossing is badged/counted a **graduation only when the profile actually changed** — the prior score's `profile_hash` differs from the current run's. On the reassess side, `get_scored_for_reassess` now also returns each posting's `prior_profile_hash` (the latest `score_event`'s hash, correlated subquery) and `reassess` gates the graduation on `prior is not None and prior != profile_hash`; a same-profile crossing folds into `unchanged`, never announced. On the digest side, `get_scored_shortlist` computes `prior_profile_changed` (latest vs immediately-prior `score_event` hash) and `ShortlistItem` carries it into `_is_graduation`. An unknown/missing prior hash (a pathological pre-0004 row) can't *prove* a change ⇒ not a graduation. Change 1 already removes most false graduations by stabilizing boundary scores; this is the belt-and-suspenders semantic guarantee.

## Alternatives Considered
- **The M7 shadow-`code_total` cut-over** (the pre-committed next step). Rejected *with evidence*: the code-total inherits the LLM subscore noise (max spread 71 > the holistic's 60), only ~2 days of thin data exist, and there is **zero ground truth** to calibrate toward (`application_event` = 0 rows, `score_override` = 1/286). A re-weight can't fix subscore noise. Parked.
- **Averaging N samples into one number.** Rejected: an averaged score is a frankenscore — its narrative/subscores no longer match its number. The median *sample* keeps a coherent judgment.
- **Always resample every job (higher N).** Rejected (P1 + cost): the noise only *matters* at the boundary; clearly-in/clearly-out jobs don't change membership, so paying N× on them is waste. The margin guard spends only where the decision is actually contested.
- **A new required `SearchSpec` knob for N/margin.** Rejected: it would be a config-first breaking change (every deployed config would fail `from_yaml` load) for a technical anti-noise default users shouldn't have to set. Module constants + defaulted params instead.
- **Self-consistency via a deterministic second scorer / lower temperature.** Rejected: temp is already 0; the residual non-determinism is the model's, not a sampling-temperature artifact — resampling the same call is the honest measure.

## Consequences
- **Easier:** the shortlist boundary stops being a coin-flip — a job near the cut is decided by the median of 3 samples, not one lucky draw; and a "graduation" now means a real skill gain, not LLM jitter (the reassess feature's whole promise, [ADR-0023](0023-reassess-replay.md), is restored to honesty).
- **Cost (bounded, documented):** ~62/286 boundary jobs get +2 scoring calls per run (~pennies on `v4-pro`); clearly-in/clearly-out jobs and `N=1` are unchanged. A resampled posting logs its shadow `code_total` N× (log noise only — a noted minor).
- **Deadline-safe:** resampling never pushes a run past the H-2 wall (extra samples stop at the deadline; the run returns `partial`, as before).
- **Observability:** reassess already reports the `delta_buckets`/`max_delta`/`mean_delta` drift meter ([ADR-0028](0028-scorer-subscores-shadow.md)); the live-validation target is that the boundary spread shrinks vs the ~16-pt baseline with **no same-profile graduations**.
- **Live-validation (pending the deploy checkpoint):** one reassess run over the deployed stack must show the boundary spread shrink and zero same-profile "graduations" before the release tag (`v0.10.1`/`v0.11.0`).
- **Parked (named, not dropped):** M7 code-total cut-over; the dark human-judge feedback loop (0 outcomes — the reason there's no calibration target); B-2 deliverability (Gmail filter is a user action, DKIM blocked on a domain); the silent-`500` alarm gap (a log-metric-filter companion).
