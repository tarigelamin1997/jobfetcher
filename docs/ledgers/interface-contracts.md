# Ledger · Interface Contracts (Produces → Consumes)

> The single source of truth for what each stage **emits** and who **consumes** it. When a release closes, verify its *Consumes* against already-shipped *Produces*, then append its *Produces* row. This prevents cross-stage drift in a multi-stage pipeline — and doubles as lineage documentation.

**Status:** nothing shipped (tagged) yet. Several contracts are **built** (🚧): the **`SearchSpec`** search input, the **`DissectedPosting`** silver dissection contract (build unit C-1), with build unit **Unit A** the **`Profile`** contract + the **gold-filter** Produces, with **Unit B** the **`score`** contract (the Scorer), and with **Step 6** the **`notify`** Produces (the daily SES digest — and ADR-0015's previously-undefined **`Notifier`** port is now defined). The rest below are the *planned* v0 contracts (⬜ until v0.1 ships).
